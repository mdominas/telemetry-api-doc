Telemetry API - V2

Simple API doc for Telemetry API endpoints.

API Endpoints

All API endpoints marked as “SECURE” require valid “token” parameter.

Acquisitions

API endpoints for Acquisitions.

Enter (PUBLIC)

Saves user’s competition (or promotion) entry in Telemetry.

REQUEST

POST

/acquisitions/{id}/enter
Parameters
id int

Acquisition ID
verticalId int

Vertical ID
email string

User’s email address
firstname string

User’s first name
lastname string

User’s last name
addressLine1 string

User’s address line 1
addressLine2 string

User’s address line 2
city string

User’s city
postcode string

User’s postal code
telephone string

User’s telephone
answerId int

Answer Id (required for promotion entries)
optins array

Array of 3rd party opt-in IDs
Headers
Content-Type application/json
application/json
BODY SCHEMA 
                          {
    "verticalId":1,
    "email":"michaeljordan@gmail.com",
    "firstname":"Michael",
    "lastname":"Jordan",
    "addressLine1":"X House Name",
    "addressLine2":"Street Name",
    "city":"City",
    "postcode":"Postcode",
    "telephone":"001122334455",
    "answerId":1,
    "optins":[1,2]
}

RESPONSE

Headers
Content-Type application/json
200 application/json
BODY SCHEMA 
                        {
    "status":200,
    "data":"OK"
}

Top level Team ID
firstname string
